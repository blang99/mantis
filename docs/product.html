The composer

Four intents. One always active.

A single segmented selector — Build is the default. Pick an intent and the input self-documents what it will do.

Build · default

Build

Describe a design in plain English. MANTIS plans it, streams the reasoning, then places native Grasshopper components on your canvas — wired, grouped by stage, and labelled. Click any plan step to frame its group.

Agent

Agent

A design assistant across both Rhino and Grasshopper. It creates and edits geometry, runs Rhino commands, and builds definitions step by step — recording undo points and asking before anything destructive.

Ask

Ask

A conversational answer about Grasshopper or your current canvas. MANTIS reads what's there and explains or advises — no building. The fastest way to learn the parametric move you're missing.

Iterate · Multi

Iterate & Multi-Solution

Iterate builds on top of the existing graph — add panelization, floor thickness, mullions, nothing dropped. Multi-Solution generates 2–3 alternative approaches so you can compare before committing.

Image → geometry

Drop an image.
Get precise geometry.

Hand MANTIS a reference photo, a sketch, or a drawing with a dimension on it. A vision model reads the form and the numbers — a "6000" callout becomes a 6000 mm reference — then builds a parametric definition scaled to match. Not a traced mesh: a real, editable graph.

# facade_elevation.jpg →
read   grid 12 × 8 bays, mullion @ 1500mm
scale  dim callout "6000" → 6000mm
build  surface → bay grid → mullion sweep
verify 96 panels · 0 warnings · editable
Under the hood

It reads the canvas it just built.

◉

Streamed reasoning

A folded plan emits before the JSON — you see how MANTIS stages the graph and why, in one streamed call. Educational, not decorative.

⟲

Self-healing

After building, it reads back live component values, detects warnings and type mismatches, and repairs them — then reports a verdict.

⌖

Click-to-jump plan

Each plan step maps to a labelled group on the canvas. Click a step and MANTIS frames that exact cluster of components.

▤

Project memory

It remembers what worked in your file — a lessons flywheel that makes later builds in the same project sharper.

⚑

Honest verdicts

Runs end with a chip: green "verified" or amber "issues remain." You always know the health of what it built.

⌦

Always undoable

Every action records an undo point. Destructive operations ask first. Nothing happens to your model you can't reverse.

Provider-agnostic

Bring any model. Or none at all.

Five providers, latest generation each — or run fully offline.

Claude

Sonnet & Opus over HTTP, or the Claude CLI for local-key streaming with vision.

OpenAI

GPT-4.1, GPT-4.1 Mini, o3-mini — JSON-mode reliable, vision-capable.

Gemini

2.5 Flash (free) and 2.5 Pro, 1M-token context for big catalogs.

OpenRouter

One key, many models — a curated shortlist of the strongest builders.

Ollama

Fully local, no key, no network. Your prompts never leave the machine.

🔒

Your choice, always

Switch providers per-task from the panel. No MANTIS account, no telemetry gate.
