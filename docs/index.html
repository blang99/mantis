The Workflow

Three ways MANTIS works the canvas

It doesn't just dump components. It reasons out the build, wires it, groups it by intent, and repairs it — the way an expert would.

02 — Wire & Group

Watch it think — then watch it organize.

Before a single wire is drawn, MANTIS explains its plan stage by stage. Then it wires the graph and wraps each logical stage in a labelled, colour-tinted group — so the canvas reads like an expert built it, not a machine.

  • Thought-process narration — the reasoning behind every stage
  • Auto-grouped workflow — tinted, named stage boxes on canvas
  • Readable data flow — understand it, edit it, learn from it

◆ Thought Process
1. Parameters — three sliders expose base radius, top radius and floor count so the whole tower stays adjustable.
2. Base Geometry — a Circle CNR turns the base radius into the ground-floor plate that feeds the array.
3. Vertical Array — Series + Move stack the floors; Loft skins them into a single tapering surface.
03 — Self-Heal

Errors that resolve themselves.

A red component never makes it to you unnoticed. MANTIS reads the runtime error, traces the data flow that caused it, and offers a one-click fix — and it back-fills sensible defaults so the graph resolves clean, with zero warnings.

  • Reads real runtime errors — not guesses
  • Auto-fills required inputs — ships a working file every time
  • One-click apply — wrapped in undo, always reversible
